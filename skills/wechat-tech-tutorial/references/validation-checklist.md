# 内容校验清单（强制执行）

**本清单用于步骤④和步骤⑤，必须逐项检查并输出结果。**

---

## 一、代码验证

### 检查项

```markdown
□ 1. 所有代码示例是否完整可运行？
   - 无"省略号跳过"
   - 无"此处省略 XXX 行"
   
□ 2. 依赖版本是否明确标注？
   - 如：`pip install transformers==4.37.0`
   - 如：`ollama pull qwen3.5:7b`（指定版本）
   
□ 3. 错误处理是否完整？
   - try-catch 包裹
   - 边界条件检查
   - 输入验证
   
□ 4. 是否标注预期输出结果？
   - 示例输出
   - 日志信息
```

### 输出格式

```markdown
### 代码验证结果

**已验证代码：**
1. [代码片段名称] - ✅ 已测试通过
2. [代码片段名称] - ✅ 已测试通过

**依赖版本确认：**
- 包名==版本号（来源：官方文档 URL）

**待确认：**
- [ ] XXX 代码需要在实际环境测试
```

## 二、事实验证

### 检查项

```markdown
□ 1. 每个技术断言是否有来源支撑？
   - 版本号 → 官方文档
   - 性能数据 → 测试报告或官方数据
   - 发布日期 → 官方发布页
   
□ 2. API 签名、参数、返回值是否核对？
   - 逐一对照官方文档
   - 无凭记忆复述
   
□ 3. 是否多源交叉验证（≥3 个独立来源）？
   - 官方文档
   - GitHub Issues
   - 技术社区讨论
```

### 输出格式

```markdown
### 事实验证结果

**关键信息验证：**
| 信息 | 来源 1 | 来源 2 | 来源 3 | 结论 |
|------|--------|--------|--------|------|
| 版本号 | 官方文档 | GitHub | 社区 | 一致 ✅ |
| 显存占用 | 官方 | 测试报告 | - | 需标注测试环境 |

**修正记录：**
- [修正 1] 原"Qwen3.5" → 修正为"Qwen3.5-Plus（2026 年 2 月版本）"
- [修正 2] 原"显存占用 500MB" → 修正为"约 500MB（M3 Pro，4bit 量化）"
```

## 三、逻辑性审查

### 检查项

```markdown
□ 1. 因果关系是否有依据？
   - "因为 A 所以 B" → A 是否真实导致 B？
   - 不是相关性当因果
   
□ 2. 数据是否前后一致？
   - 显存占用前后是否矛盾？
   - 速度数据是否一致？
   
□ 3. 是否说明适用场景和不适用场景？
   - "推荐 X 场景"同时说明"不推荐 Y 场景"
   
□ 4. 是否避免绝对化表述？
   - "一定"、"绝对"、"100%" → "通常"、"大多数情况"
   
□ 5. 是否承认知识边界？
   - "截至 2026 年 3 月"而非绝对表述
```

### 输出格式

```markdown
### 逻辑性审查结果

**因果关系确认：**
- ✅ "因为 4bit 量化，所以显存占用降低" → 有依据
- ⚠️ "速度快所以更好" → 需补充"在 X 场景下"

**数据一致性：**
- ✅ 显存占用：全文统一为"约 500MB（M3 Pro，4bit）"
- ✅ 速度数据：标注测试环境

**场景覆盖：**
- ✅ 推荐场景：本地开发、小规模测试
- ✅ 不推荐场景：生产环境高并发

**绝对化表述检查：**
- ✅ 无"一定"、"绝对"等表述
```

## 五、去 AI 味检查

### 检查项

```markdown
□ 1. 删除"总的来说"、"综上所述"、"值得注意的是"
□ 2. 删除"首先/其次/最后"（改用小标题）
□ 3. 删除"希望对你有所帮助"、"仅供参考"
□ 4. 删除"作为 AI..."自我指涉
□ 5. 观点明确，不永远"各有优劣，视情况而定"
□ 6. 有具体数字、版本、场景
□ 7. 代码有错误处理，不是完美示例
□ 8. 语气像技术博客，不是说明书
```

### AI 味 vs 人味 对比

| AI 味 ❌ | 人味 ✅ |
|---------|--------|
| "总的来说，Qwen3.5 是强大的模型" | "Qwen3.5 在 32K 上下文内表现稳定" |
| "值得注意的是，显存占用因硬件而异" | "M3 Pro 16GB 上约 500MB，Intel i7 上约 800MB" |
| "各有优劣，视情况而定" | "我推荐 X：简单场景够用，调试方便" |
| "希望本教程对你有所帮助" | "有问题评论区见，踩坑了记得告诉我" |

### 输出格式

```markdown
### 去 AI 味检查结果

**已删除/修改：**
- [ ] "总的来说" → 删除
- [ ] "值得注意的是" → 改为具体描述
- [ ] "希望对你有所帮助" → 改为"有问题评论区见"

**已添加人味：**
- [ ] 具体数字：显存占用、速度数据
- [ ] 个人判断："我推荐 X，因为..."
- [ ] 踩坑经验："我在这遇到过 XXX 问题"
```
